| [PaperSearch Research Assistant](https://github.com/sjordan1975/papersearch-research-assistant) | I built a RAG pipeline to query academic papers, using 50 arXiv papers and 488 human-written queries from the Open RAG Benchmark. I tested different chunking methods (fixed, recursive, semantic), embedding models (SentenceTransformers MiniLM-L6 and mpnet-base, both run locally with no API cost), and retrieval approaches (dense/FAISS, BM25, and hybrid with alpha-weighted fusion) in 15 setups. Adding cross-encoder reranking (ms-marco-MiniLM-L-6-v2) improved MRR from 0.733 to 0.789, a 7.6% increase, without changing the index. This did double the latency, but I still met the quality goals. | [Dating Compatibility Embedding System](https://github.com/sjordan1975/dating-compatibility-finetuning) | I fine-tuned a sentence transformer with contrastive loss to predict relationship compatibility from dating profile text, showing that compatibility is not the same as textual similarity. The pre-trained model performed poorly (AUC 0.40, worse than random), but after fine-tuning, performance improved to AUC 0.91 with Cohen's d = 2.17, which is a very large effect size. After seven rounds of tuning hyperparameters like epochs, learning rate, batch size, and warmup, accuracy plateaued at 83%. This suggests that the limit comes from the data itself, not the training setup. The main takeaway is that generic profile language, such as "I value honesty," makes it hard for the model to learn useful patterns. To improve further, we need more specific training data rather than more engineering. <br><br>Python · SentenceTransformers · UMAP · HDBSCAN · Pydantic · Matplotlib · pytest |
| [Digital Clone: Multi-Agent Communication System](https://github.com/sjordan1975/digital-clone-multi-agent) | I built a five-agent LangGraph pipeline that learns how a specific employee communicates from real email history and answers knowledge-based queries in their style, routing low-confidence questions to a human via calendar booking. The system extracts 11 stylometric features from 2,156 Enron emails, builds a 530K-chunk FAISS index from CS textbooks, and evaluates responses across three dimensions: style (cosine similarity), groundedness (LLM-as-judge), and confidence (algorithmic). The main finding is that post-hoc style rewriting made scores worse, not better — the raw output scored 0.678 on style versus 0.565–0.599 with the rewriter. This confirmed an architectural constraint: generating grounded technical content and rewriting it into a business executive's terse email voice are competing objectives at the feature level.
